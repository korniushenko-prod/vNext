Boiler Abstraction Map v1

Эта страница отвечает на вопрос “что где должно жить” на примере котла. Она не заменяет Boiler Flow View v2, а объясняет, из каких слоёв тот flow вообще собран: что является физикой, что примитивом, что sequence, что модулем и что уже whole-boiler composite.

Основной пользовательский слой: Modules Сценарии и фазы: Sequences Примитивы: Blocks Физика и транспорт: Channels

Лестница абстракций

Идея простая: чем выше слой, тем меньше пользователь видит внутреннюю механику и тем больше он настраивает понятную функцию.

От примитива к механизму
1. Channels Физика и transport DI/DO, relay, analog input, Modbus register, external resource.
2. Signals Нормализованные точки `fuel_temp.value`, `fan.running`, `button.short_press`.
3. Blocks Примитивы timer, latch, selector, hysteresis, threshold, freshness.
4. Sequences Фазы и переходы purge, ignition, run, cooldown, post-purge.
5. Modules Пользовательские функции Level Control, Flame Supervision, Fuel Mode, PID Heater.
6. Composite Котёл целиком Whole-boiler mechanism с несколькими параллельными lane.

Как это стыкуется с Boiler Flow View v2

`Flow View v2` показывает работу котла как параллельные lane. Эта страница объясняет, откуда те lane берутся внутри архитектуры.

Stack → Flow mapping

→
Signals

Blocks
hysteresis, threshold, interlock
→
Modules
Level Control, Fuel Mode, Stop Policy
→
Flow View v2
lane котла

Практический смысл этой связки

  • Flow View v2 не должен показывать raw blocks как главный язык пользователя.
  • В Flow View пользователь видит модули и policy lane, а не отдельные latch/selectors/timers.
  • Если нужно глубже, он проваливается: Flow → Module → Advanced → Sequence / Blocks.
  • Один и тот же flow-узел может опираться сразу на несколько lower-layer объектов, но снаружи остаётся одной понятной функцией.

Какие правила из этого следуют для UI

Это уже прямые продуктовые последствия для будущего конструктора.

UI rules
Blocks Оставить примитивами Простой timer, latch, selector, threshold, hysteresis. Без giant user-facing wizard.
Sequences Оставить фазами Purge, ignition, run, cooldown, stop-watch, post-purge.
Modules Сделать главным UX Пользователь настраивает функцию, а не raw helper chain.
Composite Показывать механизм Котёл как набор parallel lane, а не как один giant sequence.
