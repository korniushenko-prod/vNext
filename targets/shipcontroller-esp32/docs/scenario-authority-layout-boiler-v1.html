Boiler Layout: Context + Sources + Policy + Logic + Actuators

Это пример того, как раскладывать сценарий котла так, чтобы `ON/OFF`, `AUTO/MANUAL`, `LOCAL/REMOTE` и прочие режимы не смешивались с датчиками и выходами в одну кашу. Здесь есть отдельный верхний слой управления и 4 понятные зоны ниже.
Сверху: Control Context Слева: Sources Центр-лево: Authority / Policy Центр-право: Logic / Sequences Справа: Actuators

Этот слой отвечает не на вопрос “что происходит в процессе?”, а на вопрос “кто вообще имеет право сейчас управлять и разрешён ли запуск”.

Почему это понятнее

  • `AUTO/MANUAL` и `ON/OFF` не смешиваются с датчиками.
  • Сразу видно, кто источник, кто policy, кто sequence и кто исполнитель.
  • Можно вставить новый узел “между” слоями без хаотичного canvas.
  • Одинаково подходит и для boiler, и для другой автоматики.

Что это значит для продукта

  • Верхний уровень UI должен быть scenario-first.
  • Внутри workspace лучше держать не просто `Inputs -> Logic -> Outputs`, а `Sources -> Authority/Policy -> Logic/Sequences -> Actuators`.
  • `Main` остаётся root-сценарием, а child sequences живут в колонке Logic.
  • Blocks остаются внутренним/advanced уровнем и не заменяют этот layout.
